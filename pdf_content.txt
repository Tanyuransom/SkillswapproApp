ABSTRACT  
SkillSProf  is a Flutter mobile platform enabling students to teach and learn skills from peers. 
Features include onboarding, authentication, dashboard with categories, skill discovery, course 
enrollment, messaging, payment integration, profile management, and content upload. Built with 
Firebase microservices architecture and implementing clean architecture patterns (Repository, 
Singleton, Factory, Observer), the platform ensures scalable, maintainable peer learning.

CHAPTER 1: INTRODUCTION  
1.1 Overview  
SkillProf  connects student educators with learners through a mobile platform featuring skill 
categories (Learn Skills, Teach Skills), course discovery with ratings, payment integration 
(MTN/Orange Money), real -time messaging, and content management. The platform includes 
25+ screens covering onboarding, authentication, learning, teaching, communication, and 
transactions.  
--- Page 3 ---
1.2 Problem Statement  
Students face expensive courses, unverified teachers, limited payment options, and poo r 
communication tools. SkillProf  addresses these through skill categorization, integrated payments, 
real-time chat, and rating systems.  
1.3 Objectives  
1. Skill Discovery:  Category -based browsing (Learn/Teach Skills)  
2. Course Management:  Upload lessons (PDF/Video), set pricing  
3. Payments:  Mobile money integration (MTN, Orange)  
4. Communication:  Real-time inbox messaging  
5. Quality:  Rating/review system, blog discovery  
 
1.4 Platform Purpose  
 SkillProf serves as a bridge between student educators and learners, democratizing access to 
quality education while creating economic opportunities for skilled students. By maintaining 
rigorous verification standards and quality controls, the platform builds trust within the learning 
community and ensures sustainable, high -quality educational experiences.  
 
1.5 Key Features  
• Onboarding:  3 welcome screens with skip/next navigation  
• Authentication:  Sign up (student/tutor), sign in, password recovery, verification codes  
• Dashboard:  Categories (Learn/Teach), trending offers, recommendations, filters  
• Discovery:  Search with voice input, category filtering, skill cards with pricing/ratings  
• Courses:  Detailed views with outlines, pricing (100,000fr - 150,000fr), "Pay to learn" 
button  
• Payments:  MTN/Orange Money selection, checkout with tax calculation  
• Messaging:  Inbox with search, personal conversations with timestamps  
• Profile:  Edit profile, payment methods, language, order history, help center  
• Content Upload:  PDF/Video upload, gallery/folder selection, post completion  
• Notifications:  Settings for payments, sounds, vibrate, offers, cashback, promos  
• Blogs:  Discovery dashboard with featured articles, categories  
 
1.6 Innovations:  
--- Page 4 ---
1. AI-Powered Verification:  Unlike traditional platforms that rely solely on credential 
uploads, SkillProf  incorporates AI -generated examinations that assess practical 
knowledge and competency, providing an accessible alternative for users without formal 
certifications.  
2. Trust -Based Architecture:  The platform's verification requirements create a trusted 
ecosystem where learners can confidently invest time and resources knowing instructors 
have demonstrated genuine expertise.  
3. Community -Driven Quality:  The integrated rating system creates a self -regulating 
marketplace where quality naturally rises to prominence and poor performers are 
identified and addressed.  
4. Microservices Flexibility:  The backend architecture separates concerns into distinct 
services (Auth, User, Verification, Lesson, Messaging, Rating, Notification), enabling 
independent scaling and maintenance of critical platform functions.  
 
1.7 Target Audience  
Primary:  Students (18 -30 years) seeking affordable skills or monetization  
Secondary:  Tutors offering specialized knowledge

Architecture Description  
The system is designed using a Microservices Architecture with Firebase Backend -as-a-
Service (BaaS) . This architecture follows an Event -Driven Microservices approach combined 
with the API Gateway pattern , enabling scalability, real -time communication, and loose 
coupling between system components.  
The mobile application acts as the client layer , providing user interaction and communicating 
with backend services through a centralized API Gateway . The API Gateway serves as the 
single entry point for all client requests, handling authentication, authorization, request 
validation, and routing to the appropriate microservices.  
Each core functionality of the system is implemented as an independent microservice , such as 
Authentication, User Profile, Course Management, Skill Discovery, Messaging, Payment 
Processing, Notifications, Content Management, and Blog Services. These microservices are 
loosely coupled and independently deployable, allowing the system to scale efficiently and 
support continuous development.  
The architecture is event -driven , meaning that services communicate not only through 
synchronous REST APIs but also through asynchronous events. Firebase Cloud Functions and 
Firestore triggers are used to listen for system events such as user registration, course enrollment, 
content upload, payment completion, and message delivery. When an event occurs, the relevant 
services react automatically without direct service -to-service dependency.  
Firebase provides the Backend -as-a-Service layer , offering managed services including 
Firebase Authentication for secure user access, Firestore for real -time data storage, Cloud 
Storage for media files, Cloud Functions for serverless business logic, and Firebase Cloud 
Messaging for push notifications. This reduces infrastructure management overhead while 
ensuring high availability and performance.  
External services such as payment gateways  and AI services  are integrated through secure APIs 
and triggered via backend services when required. The use of Firebase ensures real -time 
synchronization between the backend and client application, providing instant updates to users.

Overall, this architecture ensures modularity, scalability, real -time responsiveness, and 
maintainability , making it suitable for a modern mobile learning platform with dynamic user 
interaction and future extensibility.

Pros and Cons of the Architecture  
Advantages:  
     Scalability:  
• Independent service scaling based on demand  
• Verification Service can scale during exam periods without affecting other services  
• Automatic scaling handled by Firebase  
     Maintainability:  
• Clear service boundaries reduce complexity  
• Changes to one service don't require redeploying others  
• Each service has focused responsibility  
     Fault Isolation:  
• Failure in Messaging Service doesn't crash Lesson Service  
• Circuit breaker patterns prevent cascading failures  

--- Page 20 ---
• Graceful degradation possible  
     Technology Flexibility:  
• Each service can use appropriate technology  
• Can replace services independently (e.g., swap AI provider)  
• Easy to add new services  
     Development Velocity:  
• Small team can work on services independently  
• Parallel development of features  
• Faster deployment cycles  
     Cost Efficiency:  
• Pay-per-use model (Cloud Functions)  
• No server maintenance overhead  
• Generous Firebase free tier for MVP  
     Real -Time Capabilities:  
• Firestore  listeners enable real -time chat  
• Instant notification delivery  
• Offline support built -in 
     Security:  
• Defense -in-depth (Firewall, Auth, Security Rules)  
• Managed security updates from Firebase  
• Built -in DDoS protection  
Disadvantages:  
    Complexity:  
• More complex than monolithic architecture  
• Requires understanding of distributed systems  
• Debugging across services can be challenging  
    Network Latency:  
• Service -to-service calls add latency  
• More network hops than monolithic  
--- Page 21 ---
• Mitigation: Minimize cross -service calls, use caching  
    Vendor Lock -in: 
• Heavy dependence on Firebase ecosystem  
• Migration to other providers would be costly  
• Mitigation: Use abstraction layers (Repository pattern)  
    Testing Complexity:  
• Integration testing requires multiple services running  
• End-to-end testing more complex  
• Mitigation: Use Firebase emulator suite for local testing  
    Data Consistency:  
• Eventual consistency in distributed system  
• Race conditions possible in concurrent updates  
• Mitigation: Use Firestore transactions for critical operations  
    Monitoring Overhead:  
• Need to monitor multiple services  
• Log aggregation across services  
• Mitigation: Firebase Console provides centralized monitoring  
    Cold Start Latency:  
• Cloud Functions experience cold starts (1 -2 seconds)  
• Affects first request after period of inactivity  
• Mitigation: Keep functions warm, optimize startup time
